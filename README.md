# 🔒 LanChat — Secure Enterprise-Ready Messenger

![C++](https://img.shields.io/badge/C%2B%2B-23-blue.svg)
![Qt](https://img.shields.io/badge/Qt-6.0%2B-green.svg)
![PostgreSQL](https://img.shields.io/badge/PostgreSQL-14%2B-blue.svg)
![License](https://img.shields.io/badge/License-MIT-lightgrey.svg)

**LanChat** — це багатофункціональний захищений консольний месенджер із клієнт-серверною архітектурою. Проєкт забезпечує високий рівень безпеки завдяки використанню SSL/TLS для передачі даних та криптографічного шифрування повідомлень безпосередньо в базі даних (Data at Rest).

Система підтримує ізольовані кімнати (публічні та приватні), особисті повідомлення (Direct Messages), керування історією чату та систему підрахунку непрочитаних повідомлень.

## 🚀 Основні можливості

* 🛡️ **Подвійний захист:**
    * Транспортний рівень захищений через `QSslSocket` (SSL/TLS).
    * Дані в базі шифруються за допомогою симетричного алгоритму розширення `pgcrypto`.
    * Паролі хешуються за алгоритмом SHA-256.
* 💬 **Гнучка система кімнат:**
    * **Public:** Відкриті кімнати для всіх користувачів.
    * **Private:** Закриті кімнати за запрошеннями.
    * **Direct:** Автоматично створювані кімнати для особистих переписок (через `@login`).
* ✏️ **Керування повідомленнями:** Можливість редагувати власні повідомлення, закріплювати/відкріплювати (`pin/unpin`) важливу інформацію та завантажувати стару історію чату.
* 🔔 **Непрочитані повідомлення:** Сервер автоматично відстежує `last_read_message_id` і показує кількість нових повідомлень у кожній кімнаті.
* 📝 **Сучасний CLI Інтерфейс:** Зручна система команд (Command Registry) зі вбудованою довідкою (`/help`).
* 📜 **Логування:** Інтегрована бібліотека `spdlog` для структурованого логування на стороні сервера.

## 📂 Структура проєкту

* `Common/` — Спільна статична бібліотека (команди, мережеві утиліти, базові класи).

    * `Database/` — Менеджер та репозиторії бази даних (Users, Rooms, Messages).
* `Client/` — Клієнтський додаток (CLI-інтерфейс, парсинг аргументів, підключення).

## ⚙️ Розгортання та Запуск

### 1. Вимоги (Prerequisites)
Перед збіркою переконайтеся, що у вас встановлені:
* **CMake** та компілятор з підтримкою C++23.
* **Qt 6** (з модулями Network, Sql).
* **PostgreSQL** сервер.
* **OpenSSL** (для генерації сертифікатів).

### 2. Налаштування Бази Даних
Створіть базу даних у PostgreSQL. Сервер автоматично створить усі необхідні таблиці під час першого запуску, але вам потрібно переконатися, що у вашій БД включено криптографічне розширення:
```sql
CREATE DATABASE lanchat_db;
\c lanchat_db
CREATE EXTENSION IF NOT EXISTS pgcrypto;
```

### 3. Збірка проєкту
```bash
git clone [https://github.com/nire-lisse/LanChat.git](https://github.com/nire-lisse/LanChat.git)
cd LanChat

mkdir build && cd build
cmake ..
cmake --build .
```

### 4. Налаштування SSL сертифікатів
Для роботи сервера необхідні сертифікати шифрування. Згенеруйте самопідписаний сертифікат у папці, з якої будете запускати сервер (наприклад, у теці `build/Server/`):
```bash
openssl req -x509 -newkey rsa:4096 -keyout server.key -out server.crt -days 365 -nodes
```

### 5. Конфігурація Сервера (`config.ini`)
У тій же директорії, де лежить виконуваний файл сервера, створіть файл `config.ini` з налаштуваннями підключення до бази даних та секретним ключем шифрування:
```ini
[Database]
Host=127.0.0.1
Port=5432
Name=lanchat_db
User=postgres
Password=your_db_password

[Security]
# Цей ключ використовується для шифрування повідомлень в БД за допомогою pgcrypto
EncryptionKey=SuperSecretKey123!
```

### 6. Запуск застосунків

Запуск сервера:
```bash
./LanChatServer
```

Запуск клієнта (підключення здійснюється за вказаною IP-адресою та портом):
```bash
./LanChatClient
```
*Примітка: При першому підключенні до локального сервера з самопідписаним сертифікатом можливі попередження SSL, які клієнт вміє ігнорувати.*

## 🎮 Довідник команд клієнта (CLI Commands)

Після запуску клієнта введіть `/help`, щоб побачити всі команди:

| Команда | Опис |
| :--- | :--- |
| `/login <login> <pass>` | Авторизація (або реєстрація нового користувача, якщо його не існує). |
| `/changepass <new> [old]`| Зміна пароля. (Примусова при першому вході). |
| `/rooms` | Виводить список доступних вам кімнат та кількість непрочитаних повідомлень. |
| `/join <name>` | Вхід до публічної або приватної кімнати (напр., `/join Global`). |
| `/join @<login>` | Відкриває кімнату для приватних повідомлень з іншим користувачем (Direct Message). |
| `/createroom <name> <type> [desc]` | Створює нову кімнату. `type`: 0 (Public) або 1 (Private). |
| `/invite <login>` | Додає користувача до поточної приватної кімнати. |
| `/edit <msg_id> <new_text>`| Редагування власного повідомлення. |
| `/pin <msg_id>` | Закріплення повідомлення в поточній кімнаті. |
| `/unpin <msg_id>` | Відкріплення повідомлення. |
| `/history [before_id]` | Завантаження старіших повідомлень поточної кімнати. |
